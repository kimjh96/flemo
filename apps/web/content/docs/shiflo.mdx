## How shiflo uses flemo

shiflo is a hybrid app: the UI is a web app wrapped for iOS and Android. The hard part of that
approach is making it not _feel_ like a web view — taps that push a new screen, the back gesture,
the way a detail slides in over a list. That's the gap flemo closes.

Screen-to-screen navigation in shiflo runs on flemo: pushes and pops, the left-edge swipe-back
gesture, and the transitions between screens — all driven from plain React, with the animations
compiled to CSS keyframes. The result is a hybrid app whose motion reads as native, on both
platforms, from a single web codebase.

## Why it matters

This is the proof flemo is built for: a real app, in real users' hands, on both stores — not a
playground. If you're reaching for native-like screen transitions in a React app or a hybrid
shell, shiflo is what that looks like in production.
