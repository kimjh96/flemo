## useNavigate

```ts
const navigate = useNavigate();

navigate.push("/posts/:slug", { slug: "hello" });
navigate.replace("/login");
navigate.pop(); // back one screen
navigate.pop({ count: 2 }); // back two screens, in one transition
navigate.pop({ until: "/posts/:slug" }); // back to the nearest matching screen
```

`push` and `replace` are async — they wait for the transition to start and the route to take
effect. Every `pop`/`replace`, however far it reaches, drives a single transition.

### Going back, and collapsing

`pop` and `replace` take an optional **distance** — `{ count }` (a number of screens) or
`{ until }` (a route pattern) — to reach past the top in one transition. The screens skipped over
are removed **without ever painting**, so you never see them flash by; only the screen you land on
animates.

- `pop({ count })` — pop `count` screens. Clamps to the stack depth (popping past the root lands
  on the root). `pop()` is `pop({ count: 1 })`.
- `pop({ until })` — pop back to the nearest screen below the top whose route matches the pattern,
  and **land on it** (it stays). Searches from the current screen toward the root, stops at the
  first match. No match → no-op.
- `replace(path, params, { count })` — collapse the top `count` screens into the new screen.
- `replace(path, params, { until })` — collapse down to and **including** the nearest match (the
  matched screen is replaced), putting the new screen in its place. No match → no-op.

```ts
// stack:  /  →  /library  →  /posts/a  →  /posts/b   (top = /posts/b)
navigate.pop({ count: 2 }); // → /library
navigate.pop({ until: "/posts/:slug" }); // → /posts/a (lands on the nearest /posts/:slug)
navigate.replace("/posts/:slug", { slug: "x" }, { count: 2 }); // → /, /library, /posts/x
navigate.replace("/library", {}, { until: "/library" }); // collapse back onto the library
```

`{ count }` and `{ until }` are mutually exclusive; `until` wins if both are passed.

### Options

```ts
navigate.push(
  "/posts/:slug",
  { slug: "hello" },
  {
    transitionName: "material",
    layoutId: "photo-hello"
  }
);
```

| Option           | What it does                                                                            |
| ---------------- | --------------------------------------------------------------------------------------- |
| `transitionName` | Override the default transition for this push only                                      |
| `layoutId`       | Pair with `transitionName: "layout"` on push or replace to enable shared-element morphs |

## useParams

`useParams<T>()` returns the params for the current route, typed against your `RegisterRoute`
augmentation.

```tsx
function Post() {
  const { slug } = useParams<"/posts/:slug">();
  return <h1>{slug}</h1>;
}
```

flemo merges path params and query params into one object.

```
URL: /posts/hello?ref=home
useParams<"/posts/:slug">() → { slug: "hello", ref: "home" }
```

## useStep

`useStep()` is for sub-states within the current screen. It pushes a new history entry — so the
browser back button still goes back — but **doesn't change the route**. The same `<Screen>` stays
mounted; only its params update.

A common case: a multi-step form on one screen.

```tsx
function Onboarding() {
  const { step = "name" } = useParams<"/onboarding">();
  const stepper = useStep<"/onboarding">();

  if (step === "name") {
    return <button onClick={() => stepper.pushStep({ step: "email" })}>Next</button>;
  }

  if (step === "email") {
    return <button onClick={() => stepper.popStep()}>Back</button>;
  }
}
```

| Method                | What it does                                           |
| --------------------- | ------------------------------------------------------ |
| `pushStep(params)`    | Push a new history entry with these params, same route |
| `replaceStep(params)` | Replace current history entry with these params        |
| `popStep()`           | Go back one step (same as `window.history.back()`)     |

## Choosing between push and step

| You want to…                                    | Use                             |
| ----------------------------------------------- | ------------------------------- |
| Show a different screen                         | `navigate.push`                 |
| Replace the current screen (e.g., login → home) | `navigate.replace`              |
| Close the current screen                        | `navigate.pop`                  |
| Go back several screens at once                 | `navigate.pop({ count })`       |
| Go back to a specific earlier screen            | `navigate.pop({ until })`       |
| Replace, collapsing several screens into one    | `navigate.replace(p, q, { … })` |
| Stay on the same screen but change UI           | `useStep().pushStep`            |

<Callout type="info">
  **`useStep`** keeps everything in the screen tree — same Screen instance, same Provider, same
  scroll position. **`useNavigate`** mounts a new Screen with a real transition.
</Callout>
