## Components

| Export         | Summary                                                            | Package               | More                            |
| -------------- | ------------------------------------------------------------------ | --------------------- | ------------------------------- |
| `Router`       | Root container. Renders the active screen.                         | `@flemo/react`        | [Router](./router)              |
| `Route`        | Maps a path (or paths) to an element.                              | `@flemo/react`        | [Router](./router)              |
| `Screen`       | Per-route screen container with app/nav/safe-area slots.           | `@flemo/react`        | [Screen](./screen)              |
| `LayoutScreen` | Screen variant for shared `layoutId` morphs across screens.        | `@flemo/react-layout` | [LayoutScreen](./layout-screen) |
| `LayoutConfig` | Wraps motion children with the current screen's transition timing. | `@flemo/react-layout` | [LayoutScreen](./layout-screen) |

## Hooks

| Export                      | Returns                                                     | More                                 |
| --------------------------- | ----------------------------------------------------------- | ------------------------------------ |
| `useNavigate()`             | `{ push, replace, pop, popTo }`                             | [Navigation](./navigation)           |
| `useParams<T>()`            | The current route's params (path + query merged)            | [Navigation](./navigation#useparams) |
| `useStep<T>()`              | `{ pushStep, replaceStep, popStep }` for sub-states         | [Navigation](./navigation#usestep)   |
| `useScreen()`               | Current screen meta (`id`, `isActive`, `zIndex`, …)         | —                                    |
| `useViewportScrollHeight()` | Visual-viewport scroll metrics, useful for mobile keyboards | —                                    |

## Transition factories

| Export                | When to use                                                        |
| --------------------- | ------------------------------------------------------------------ |
| `createTransition`    | Six-phase transitions (idle / enter / exit / enterBack / exitBack) |
| `createRawTransition` | Full control over every navigate state                             |
| `createDecorator`     | Four-phase decorator (initial / idle / enter / exit)               |
| `createRawDecorator`  | Decorator with full per-state control                              |

See [Transitions](./transitions) for examples.

## Built-in decorator names

`"overlay"`

## Type registries

| Interface            | Purpose                                                            |
| -------------------- | ------------------------------------------------------------------ |
| `RegisterRoute`      | Register your routes for type-safe `navigate.push` and `useParams` |
| `RegisterTransition` | Register custom transition names                                   |
| `RegisterDecorator`  | Register custom decorator names                                    |

```ts
declare module "@flemo/react" {
  interface RegisterRoute {
    /* … */
  }
  interface RegisterTransition {
    /* … */
  }
  interface RegisterDecorator {
    /* … */
  }
}
```

## Type exports

`ScreenProps` · `TransitionName` · `RegisterTransition` · `DecoratorName` · `RegisterDecorator`

## Peer dependencies

`@flemo/react` requires only `react ^19` and `react-dom ^19`. `@flemo/react-layout` adds `motion ^12`
as a peer dependency — install it when you use `<LayoutScreen>` or `<LayoutConfig>`.
